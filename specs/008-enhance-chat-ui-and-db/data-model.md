# Data Model: Enhanced Chatbot UI & Database Chapter Storage

## Entities

### Chapter
- **id**: UUID (Primary Key, generated by PostgreSQL)
- **slug**: VARCHAR(255) (Unique, used as lookup key e.g., `@site/docs/intro.md`)
- **title**: VARCHAR(255) (Extracted from first H1 or filename)
- **content**: TEXT (Markdown content of the chapter)
- **last_synced**: TIMESTAMP (When the content was last updated in DB)
- **created_at**: TIMESTAMP (When the record was first created)

## Validation Rules
1. **Slug**: MUST follow the `@site/docs/` prefix convention for consistency with Docusaurus/frontend logic.
2. **Content**: MUST NOT be empty when saved.

## Relationships
- **Chapter** 1:N **UserPersonalization** (One chapter can have many user-specific personalized versions)
- **Chapter** 1:N **Translation** (One chapter can have many translated versions in different languages)
