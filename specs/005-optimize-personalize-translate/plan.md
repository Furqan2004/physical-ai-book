# Implementation Plan: Optimize Personalization and Translation Endpoints

**Branch**: `005-optimize-personalize-translate` | **Date**: 2026-03-07 | **Spec**: [specs/005-optimize-personalize-translate/spec.md](specs/005-optimize-personalize-translate/spec.md)
**Input**: Feature specification for database caching and remote content fetching.

## Summary

Optimize the personalization and translation endpoints by implementing a database-level caching layer to store pre-generated AI content. This will reduce token usage and improve performance for repeat requests. Additionally, the system will transition from reading local markdown files to fetching content directly from the frontend deployment URL using `httpx`.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: FastAPI, httpx, asyncpg, openai-agents
**Storage**: Neon PostgreSQL
**Testing**: pytest
**Target Platform**: Linux server
**Performance Goals**: Cached retrieval < 300ms, 0 AI tokens for repeat requests.
**Constraints**: 
- Backend changes only (do not touch frontend).
- Use `httpx` for remote content extraction.
- Personalization is per-user; Translation is global (Urdu).
- Save to DB *before* returning to the user.
